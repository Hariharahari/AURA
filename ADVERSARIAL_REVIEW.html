🔴 Critical Vulnerabilities (P0)

#1: Hardcoded Database Credentials

📁 Location: main.py:24 CRITICAL

Issue: Database credentials hardcoded in source code

NEO4J_PASSWORD = "harish@12" # ← EXPOSED

🎯 Attack Vector

  • Attacker clones repository and gains immediate database access
  • Credentials permanently in git history
  • No way to rotate without code change and redeploy

💥 Impact

  • Complete Neo4j database compromise
  • Ability to modify, delete, or exfiltrate all analyzed repositories
  • Lateral movement into database infrastructure

🔧 Fix

Use environment variables from .env file:

NEO4J_PASSWORD = os.getenv("NEO4J_PASSWORD")

#2: Unrestricted CORS Configuration

📁 Location: main.py:16-20 CRITICAL

Issue: CORS allows requests from ANY domain

allow_origins=["*"], allow_credentials=True, # ← Dangerous combination

🎯 Attack Vector

  • Malicious website can make requests to AURA API
  • CSRF attacks on users with active sessions
  • JavaScript exfiltrates analysis reports to attacker domains

💥 Impact

  • Unauthorized analysis requests from any website
  • Report exfiltration and data theft
  • Botnet-scale DDoS attacks

🔧 Fix

Whitelist specific origins:

ALLOWED_ORIGINS = os.getenv("ALLOWED_ORIGINS", "http://localhost:3000").split(",") app.add_middleware(CORSMiddleware, allow_origins=ALLOWED_ORIGINS)

#3: Path Traversal in /api/images Endpoint

📁 Location: main.py:98-104 CRITICAL

Issue: No path validation on image retrieval

🎯 Attack Vector

GET /api/images/../../.env GET /api/images/../../../../etc/passwd GET /api/images/../../backend/requirements.txt

💥 Impact

  • Read arbitrary files on the system
  • .env file with API keys and passwords exposed
  • Source code disclosure
  • System configuration files readable

🔧 Fix

Validate and restrict to specific directory:

safe_name = Path(image_name).name # Filename only image_path = (IMAGES_DIR / safe_name).resolve() # Verify path is within allowed directory

Issue: Same as #3 - repo_name parameter not validated

💥 Impact

#5: Unsafe Deserialization in FAISS Loading

📁 Location: chat_agent.py:28 CRITICAL

Issue: allow_dangerous_deserialization=True enables code execution

🎯 Attack Vector

  • Attacker crafts malicious FAISS pickle file
  • Triggers deserialization via /api/chat?repo=attacker_repo
  • Python pickle allows arbitrary code execution

💥 Impact

  • Remote Code Execution (RCE) on backend
  • Complete system compromise
  • Data exfiltration, cryptomining, malware installation

#6: No Authentication or Authorization

📁 Location: main.py - All endpoints CRITICAL

Issue: Every endpoint is publicly accessible

🎯 Attack Vector

  • Anonymous attacker can enumerate all repositories: GET /api/repos
  • Trigger expensive analysis operations
  • Read all generated reports and architectural diagrams
  • Query knowledge base of any repository

💥 Impact

  • No access control or multi-tenancy
  • DDoS via resource exhaustion
  • Information disclosure
  • Unauthorized analysis of proprietary code

🔧 Fix

Implement JWT-based authentication:

from fastapi.security import HTTPBearer, HTTPAuthCredentials security = HTTPBearer() @app.post("/api/analyze") def api_analyze_repo(request: AnalyzeRequest, credentials: HTTPAuthCredentials = Depends(security)): token = credentials.credentials payload = jwt.decode(token, SECRET_KEY, algorithms=["HS256"]) # Continue only if token valid

🟠 Major Design Flaws (P1)

#12: No Neo4j Connection Pooling

Issue: Single driver instance, no connection pool management

Impact: Connection exhaustion under load → service unavailability

#13: Missing Request Timeout Handling

Issue: No timeouts on LLM calls, git clone, or database operations

Impact: Hung processes consuming resources forever

#14: No Rate Limiting

Issue: Unlimited /api/analyze requests per client

Impact: DDoS by queuing 1000 repository analyses

#15: Unbounded Report Generation

Issue: No validation on LLM output length

Impact: Disk space exhaustion, service crashes

#16: No Cleanup of Old Repositories

Issue: Analyzed repos stored forever in cloned_repos/

Impact: Unbounded disk usage growth

#17: No Backup or Recovery

Issue: No Neo4j backups, no disaster recovery plan

Impact: Complete data loss if disk fails

#18: Concurrent Repository Analysis Not Prevented

Issue: Multiple users can analyze same repo simultaneously

Impact: File corruption, inconsistent data
